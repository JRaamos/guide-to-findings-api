Manual dos Achados

Guia passo a passo para criar conteúdo no Strapi

Manual operacional para uma pessoa que nunca usou a plataforma criar uma página de ranking bem feita, revisar o conteúdo e publicar com segurança no site público.

1 Fluxo oficial dentro do Strapi Admin
7 Etapas da pesquisa à publicação
0 Publicações sem revisão humana

Projeto: Manual dos Achados

Uso: treinamento interno, operação editorial e criação de páginas públicas.

Versão: 1.0 - atualizado em 08/06/2026.

1. Ideia central antes de começar

Regra principal: tudo que for administração de conteúdo acontece no Strapi Admin. O site em Next.js é apenas o site público que mostra páginas aprovadas e publicadas.

O fluxo correto para uma página de ranking é:

Categoria Produtos Ranking IA Revisão Publicação Site

Exemplo usado neste guia: criar uma página pública chamada Top 10 Serras Mármore, acessível em /construcao/top-10-serras-marmore.

O que a pessoa precisa ter em mãos

  • Acesso ao Strapi Admin com usuário e senha.
  • Tema da página: exemplo, "serra mármore".
  • Categoria principal: exemplo, "Construção".
  • Subcategoria: exemplo, "Ferramentas elétricas".
  • Critério editorial inicial: melhor geral, custo-benefício, alternativa, uso profissional ou básico.

Onde acessar

Em ambiente local, abra http://localhost:1337/admin. Em produção, use a URL do Strapi Admin informada pela equipe técnica.

Importante: nunca publique uma página sem revisar produto, ranking, texto, SEO, FAQ e link de compra. A IA ajuda a escrever, mas não substitui a revisão editorial.

2. Preparar categoria e subcategoria

Categorias organizam o site e fazem parte da URL pública. Sem categoria ativa, a página pode não aparecer corretamente no site e no sitemap.

1

Criar ou revisar a Category

Strapi Admin > Content Manager > Collection Types > Category
  1. Clique em Create new entry ou abra uma categoria existente.
  2. Preencha name. Exemplo: Construção.
  3. Preencha ou confirme slug. Exemplo: construcao.
  4. Escreva uma description curta, com uma frase clara sobre a área.
  5. Defina status como active.
  6. Use order para controlar a ordem no site. Exemplo: 1.
  7. Salve.
2

Criar ou revisar a SubCategory

Strapi Admin > Content Manager > Collection Types > SubCategory
  1. Clique em Create new entry ou abra uma subcategoria existente.
  2. Preencha name. Exemplo: Ferramentas elétricas.
  3. Preencha slug. Exemplo: ferramentas-eletricas.
  4. Defina status como active.
  5. No campo de relação category, selecione Construção.
  6. Salve.
Campo Como preencher bem Erro comum
name Nome claro para o usuário final. Usar nomes técnicos ou abreviações internas.
slug Minúsculo, sem acento, separado por hífen. Usar espaços, acentos ou mudar slug depois de publicar.
status Use active para aparecer publicamente. Deixar inactive e esperar aparecer no site.
Sobre IDs: algumas ferramentas internas pedem categoryId e subCategoryId. Use o número exibido no registro/listagem do Strapi. Se o ID não aparecer na tela, peça à equipe técnica para informar o ID correto antes de importar produtos.

3. Buscar e importar produtos do Mercado Livre

Esta etapa cria ou atualiza produtos, links afiliados e marketplace. Use produtos reais, relevantes e com dados suficientes para o leitor decidir.

3

Buscar produtos

Strapi Admin > Mercado Livre
  1. No campo de busca, digite o termo. Exemplo: serra marmore.
  2. No campo limite, use entre 10 e 30 para começar.
  3. Clique em Buscar.
  4. Analise os resultados pelo nome, preço, imagem, marca e relevância.
4

Selecionar e importar

Strapi Admin > Mercado Livre
  1. Marque os produtos que fazem sentido para o ranking.
  2. Informe categoryId e subCategoryId.
  3. Clique em Importar selecionados.
  4. Aguarde a mensagem de sucesso.

Como escolher bons produtos

Tem imagem real e clara.
Tem preço informado.
Tem marca ou modelo reconhecível.
Combina com a categoria e subcategoria.
Não é acessório quando o ranking é de produto principal.
Não parece anúncio duplicado ou confuso.

Revisar produtos importados

Strapi Admin > Content Manager > Collection Types > Product
  1. Abra cada produto importado.
  2. Confira name, brand, model, price e imageUrl.
  3. Confirme a relação com marketplace, category e subCategory.
  4. Defina status como approved quando o produto puder entrar em ranking.
  5. Defina availability como inStock quando houver disponibilidade conhecida.
  6. Salve.
Não use produto sem link ativo. Verifique em AffiliateLink se existe affiliateUrl e se o status está active.

4. Montar o Ranking Builder

O ranking define a decisão editorial: quais produtos entram, em que ordem aparecem e qual é a recomendação principal.

5

Criar ranking

Strapi Admin > Ranking Builder
  1. Preencha title. Exemplo: Top 10 Serras Mármore.
  2. Preencha slug. Exemplo: top-10-serras-marmore.
  3. Informe categoryId e subCategoryId.
  4. Clique em Carregar produtos.
  5. Marque pelo menos 3 produtos relevantes.
  6. Defina uma position única para cada produto. Exemplo: 1, 2, 3.
  7. Clique em Salvar ranking.
  8. Anote o rankingId gerado ou exibido na listagem.

Critério recomendado para ordenar

Posição Uso recomendado Como justificar
1 Melhor escolha geral. Equilibra qualidade, preço, avaliações e confiança.
2 Melhor custo-benefício. Boa entrega pelo preço, sem perder qualidade mínima.
3 Melhor alternativa. Serve para um perfil diferente, como uso básico ou profissional.
Evite ranking sem lógica. A ordem deve contar uma história: melhor geral, custo-benefício, alternativa, profissional, básico, barato, premium ou outra intenção clara.

5. Gerar rascunho com IA

A IA cria um rascunho de Page, Seo e Faq a partir do ranking. Nada deve ser publicado automaticamente.

6

Gerar conteúdo

Strapi Admin > AI Generator
  1. Informe o rankingId criado no Ranking Builder.
  2. Clique em Gerar conteúdo.
  3. Aguarde a mensagem de sucesso.
  4. Anote os IDs retornados: pageId, seoId, faqIds e aiGenerationLogId.

O que a IA deve gerar

  • Page draft: título, resumo, introdução, conclusão e conteúdo editorial.
  • Seo draft: meta title, meta description, canonical, robots, schema e palavras-chave.
  • Faq draft: perguntas e respostas úteis para a intenção de compra.
  • AiGenerationLog: registro técnico da geração para auditoria.
Revisão obrigatória: remova qualquer ano, data, promessa, especificação ou informação técnica que não exista nos dados do produto ou no contexto editorial. Não publique texto com "melhor de 2026", "atualizado em janeiro" ou datas inventadas.

6. Revisar no Publication Workflow

Esta é a etapa mais importante para a qualidade. A pessoa revisa o conteúdo antes de aprovar e publicar.

7

Abrir a página para revisão

Strapi Admin > Publication Workflow
  1. Na lista, procure a Page gerada pela IA.
  2. Se necessário, digite o pageId no campo de carregamento.
  3. Clique para carregar a página.
  4. Revise os dados da Page, Seo e Faq.

Campos da Page

Campo O que revisar Boa prática
Title Título público da página. Claro, direto e sem ano inventado.
Excerpt Resumo curto. Explique o benefício da página em 1 ou 2 frases.
Introduction Introdução editorial. Diga para quem é o ranking e o que foi considerado.
Summary Resumo de decisão. Ajude o leitor a entender rapidamente as melhores opções.
Conclusion Fechamento com recomendação. Finalize com orientação clara por perfil de compra.

Campos de SEO

Campo Como preencher Referência
Meta Title Entre 50 e 60 caracteres quando possível. Top 10 serras mármore para comprar
Meta Description Entre 140 e 160 caracteres quando possível. Explique a seleção e o benefício da comparação.
Focus Keyword Termo principal da página. top 10 serras mármore
Secondary Keywords Separe por vírgula. Inclua variações úteis, sem exagero.
Robots Use indexFollow para página pública real. Use noIndexNoFollow para teste ou conteúdo temporário.

FAQ

  • Use perguntas que uma pessoa realmente faria antes de comprar.
  • Evite perguntas genéricas como "Qual é o melhor produto?" sem contexto.
  • Responda com clareza e sem inventar especificações.
  • Remova FAQs duplicadas ou sem utilidade.

7. Aprovar, publicar e testar

8

Salvar revisão

Strapi Admin > Publication Workflow
  1. Depois de revisar Page, Seo e Faq, clique em Salvar.
  2. Reabra a página e confira se as alterações persistiram.
9

Aprovar

Strapi Admin > Publication Workflow
  1. Clique em Aprovar.
  2. Confirme que a Page recebeu approvedAt.
  3. Confirme que o Seo ficou com status approved.
  4. Confirme que as FAQs foram aprovadas quando existirem.
10

Publicar

Strapi Admin > Publication Workflow
  1. Clique em Publicar.
  2. A página deve mudar para status = published.
  3. O campo publishedAt deve ser preenchido.

Como testar no site público

  1. Monte a URL no formato /{categorySlug}/{pageSlug}.
  2. Exemplo: /construcao/top-10-serras-marmore.
  3. Abra a URL no navegador.
  4. Confira se aparecem: título, produtos, CTAs, FAQ, conclusão e SEO básico.
A página abre no site público.
O ranking aparece com produtos em ordem correta.
As imagens aparecem ou usam fallback visual adequado.
Os botões de oferta abrem os links corretos.
FAQ e conclusão aparecem.
A página não contém datas inventadas.

8. Checklist final de qualidade editorial

Antes de considerar uma página pronta, marque todos os itens abaixo.

Categoria está ativa.
Subcategoria está ativa e ligada à categoria correta.
Produtos relevantes foram aprovados.
Cada produto tem link afiliado ativo.
Ranking tem ordem lógica e posições únicas.
Produto #1 é defensável como melhor escolha.
Texto não parece genérico ou automático.
Não há ano, data ou informação temporal inventada.
Meta title e meta description estão revisados.
Robots está correto para indexação.
FAQ responde dúvidas reais.
URL pública está correta.

9. Problemas comuns e como resolver

Problema O que verificar
A página não aparece no site. Confira se Page está published, Category está active, slug está correto e SEO não está noIndexNoFollow.
A página não aparece no sitemap. Confira Page publicada, Category ativa e Seo.robots = indexFollow.
Produto sem imagem. Confira Product.imageUrl. Se não houver imagem, o frontend usará fallback visual.
Botão de oferta não abre. Confira se existe AffiliateLink.affiliateUrl e se o link está active.
IA gerou informação errada. Edite no Publication Workflow antes de aprovar. Se for grave, gere novamente a partir do ranking revisado.
Ranking parece fraco. Revise produtos, posições, destaques e critérios. Um ranking bom precisa explicar por que cada item está ali.
Este guia descreve o fluxo operacional atual do Manual dos Achados: Strapi Admin para administração e Next.js para site público. Em caso de dúvida, não publique: salve como rascunho e peça revisão editorial ou técnica.
